#### Система пакетных обновлений  
  Длительность пакета определяется количеством неактивных сущностей - высчитывается базовое время на одну сущность и умножается на количество сущностей.  
  Длительность пакета должна быть такой, чтобы ФПС был стабилен при динамически изменяющейся длительности пакета.  
#### Грубая симуляция с низкой частотой  
  Производится один раз в пакет.  
  Происходит симуляция деятельности сущностей, с учётом текущего поведения сущности и длительности пакета.  
#### Приоритеты обновления сущностей
  Чем ближе сущность к игроку - тем приоритетнее обновление.  
  Сущности в радиусе загрузки игрока (pR) являются активными и обновляются каждый кадр.  
  Сущности в радиусе от pR до 10 х pR обновляются пакетно.  
  Сущности в радиусе от 10 х pR обновляются каждым вторым пакетом.
